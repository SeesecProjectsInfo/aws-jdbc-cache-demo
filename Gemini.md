# 🤖 Project Gemini Context: Smart DB Failover API

This document serves as high-level context for AI assistants to understand the architecture, design principles, and development workflow of the **Smart DB Failover API**.

## 🎯 Project Overview
A production-ready Spring Boot REST API demonstrating **high availability** through automatic database failover and **performance** through Redis caching. It explicitly avoids JPA/Hibernate to maintain low-level control over database interactions using Spring JDBC.

## 🧱 Core Technology Stack
- **Backend**: Java 17, Spring Boot 3.x
- **Data Access**: Spring JDBC (JdbcTemplate), HikariCP (Connection Pooling)
- **Databases**: MySQL 8.0 (Primary + Secondary nodes)
- **Caching**: Redis 7
- **Infrastucture**: Docker Compose for local orchestration

## 🏗️ Architectural Patterns

### 1. Transparent Database Failover
The application maintains two separate connection pools. The `DBFailoverManager` handles the logic of switching between them based on health.
- **Circuit Breaker**: Implemented in `CircuitBreaker.java`. States: `CLOSED` (Healthy), `OPEN` (Failover), `HALF_OPEN` (Testing Recovery).
- **Scheduled Recovery**: A background task in `DBFailoverManager` probes the primary node every 15s to facilitate auto-recovery.

### 2. Cache-First Strategy
The `CacheService` orchestrates interaction with Redis.
- **Read Logic**: Redis Hit -> Return; Redis Miss -> DB Query -> Update Redis -> Return.
- **Write Logic**: Write to DB -> Evict stale keys from Cache.
- **Fault Tolerance**: If Redis is down, the system gracefully degrades to DB-only mode.

## 📁 Key Source Locations
- `com.smartdb.failoverapi.failover`: Core failover and circuit breaker logic.
- `com.smartdb.failoverapi.cache`: Redis integration logic.
- `com.smartdb.failoverapi.repository`: Pure SQL JDBC repositories.
- `src/main/resources/static/index.html`: Dashboard for real-time monitoring.

## 🛠️ Development Guidelines
- **No JPA/Hibernate**: Use `JdbcTemplate` for all database interactions.
- **Health-Aware Queries**: Always obtain the `JdbcTemplate` via `failoverManager.getActiveJdbcTemplate()`.
- **Failover Metrics**: Increment appropriate counters in `FailoverMetrics` when state changes occur.
- **Thread Safety**: Ensure all failover manager states are thread-safe (currently handled via `AtomicReference` and `AtomicInteger`).
